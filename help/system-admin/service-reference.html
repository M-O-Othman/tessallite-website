Service Reference

What this covers

Per-service details for all six Tessallite services: default ports, health endpoints, inter-service dependencies, restart behavior, and log format. Followed by a service summary table and a dependency graph description.

gateway

Purpose: Public entry point for BI tool query traffic. Translates JDBC and XMLA queries into an internal format and forwards them to query-router.

  • Ports: 5433 (JDBC, PostgreSQL wire protocol), 8080 (XMLA/HTTP)
  • Health endpoint: GET /api/health on port 8080. Returns {"status":"ok"} when ready.
  • Depends on: query-router
  • Restart behavior: Stateless. Safe to restart at any time. Active JDBC connections from BI tools will drop and must reconnect — configure your BI tool's connection pool to retry on disconnect.
  • Log format: Structured JSON lines on stdout. Each line includes timestamp, level, service, msg, and request-specific fields.

query-router

Purpose: Routes translated queries either to a pre-built aggregate table or to the raw data source. Records query misses in the model-service log.

  • Port: Internal HTTP only. Not exposed to the host network.
  • Health endpoint: GET /api/health. Returns {"status":"ok"} when ready.
  • Depends on: model-service
  • Restart behavior: Stateless. Safe to restart at any time without data loss.
  • Log format: Structured JSON lines on stdout.

model-service

Purpose: The semantic layer. Serves model definitions (tables, joins, dimensions, measures, aggregates) to query-router and the frontend. Writes query misses to the internal database for optimizer to consume.

  • Port: Internal HTTP only. Not exposed to the host network.
  • Health endpoint: GET /api/health. Returns {"status":"ok","db_migration":"VERSION"} when ready. The db_migration field shows the current schema version.
  • Depends on: internal PostgreSQL (postgres)
  • Restart behavior: Stateless. All model data is persisted in PostgreSQL. Safe to restart at any time.
  • Log format: Structured JSON lines on stdout.

optimizer

Purpose: Reads the query miss log from model-service, scores aggregate candidates by ROI, and creates build recommendations that appear in the frontend.

  • Port: Internal HTTP only. Not exposed to the host network.
  • Health endpoint: GET /api/health. Returns {"status":"ok"} when ready.
  • Depends on: model-service
  • Restart behavior: Stateless. Recommendations are recalculated from the miss log on each run. Safe to restart at any time.
  • Log format: Structured JSON lines on stdout.

scheduler

Purpose: Builds and refreshes aggregate tables in the user's target schema on their data source. Executes on a cron schedule and on demand.

  • Port: Internal HTTP only. Not exposed to the host network.
  • Health endpoint: GET /api/health. Returns {"status":"ok"} when ready.
  • Depends on: model-service
  • Restart behavior: Stateless. In-progress builds may be interrupted on restart; scheduler will re-queue them on the next run. Safe to restart at any time.
  • Log format: Structured JSON lines on stdout.

frontend

Purpose: Web management interface. Provides model building, aggregate management, connection setup, user management, and platform monitoring for all roles.

  • Port: 3000 (HTTP). Exposed to the host network.
  • Health endpoint: GET /api/health on port 3000. Returns {"status":"ok"} when ready.
  • Depends on: model-service
  • Restart behavior: Stateless. All session data uses signed tokens; active user sessions may need to re-authenticate after restart.
  • Log format: Structured JSON lines on stdout.

Service dependency graph

Dependencies flow in one direction. A service will not function if any of its dependencies is unavailable:

  • gateway depends on query-router
  • query-router depends on model-service
  • model-service depends on internal PostgreSQL
  • optimizer depends on model-service
  • scheduler depends on model-service
  • frontend depends on model-service

Start order when bringing the platform up from scratch: postgres → model-service → (query-router, optimizer, scheduler, frontend) → gateway.
