Supported Data Sources

What this covers

Tessallite connects to backend data sources to read raw data and write pre-aggregated summaries. This article lists the supported sources, their drivers, connection parameters, and notes on setup and permissions.

Summary

Data source Driver included Auth method Notes
PostgreSQL Yes (bundled) Username / password Tested PostgreSQL 13+. Supports SSL.
BigQuery No (must add) Service account JSON key Simba/Google BigQuery JDBC driver required.
Hadoop / Spark Thrift Server Yes (bundled Hive driver) None or Kerberos Tested with Spark 3.x Thrift Server.

PostgreSQL

PostgreSQL is the primary supported source. The PostgreSQL JDBC driver is bundled with Tessallite — no additional installation is required.

Parameter Value
Host Hostname or IP of the PostgreSQL server
Port 5432 (default)
Database PostgreSQL database name
User PostgreSQL user with read access to source schemas
Password PostgreSQL user password
SSL Optional. Append sslmode=require to the connection parameters.

Compatible managed services: AWS RDS for PostgreSQL, Azure Database for PostgreSQL, Google Cloud SQL for PostgreSQL. Connection parameters are the same as for self-hosted PostgreSQL.

Permissions required: The database user must have USAGE on source schemas and SELECT on source tables. For aggregate writes, the user must have CREATE TABLE, INSERT, and DROP TABLE on the target schema.

BigQuery

Tessallite connects to BigQuery via the Simba (Google) BigQuery JDBC driver. This driver is not bundled and must be added to the Tessallite classpath before configuring a BigQuery source.

Parameter Value
GCP project ID Your Google Cloud project ID
Dataset name The BigQuery dataset containing source tables
Service account key Path to the service account JSON key file on the Tessallite host

Permissions required: The service account must have the bigquery.jobs.create and bigquery.tables.getData IAM roles on the project. For aggregate writes, the service account also needs write access to the target BigQuery dataset.

Query dialect: Tessallite uses standard SQL when querying BigQuery.

Driver setup: Download the Simba BigQuery JDBC driver from the Google Cloud documentation. Place the JAR file in the Tessallite driver directory (typically lib/jdbc/) and restart the Model Service.

The BigQuery JDBC driver must be installed separately. Tessallite does not distribute it. Verify the driver version is compatible with your GCP project's BigQuery API version.

Hadoop / Spark Thrift Server

Tessallite connects to Hadoop and Apache Spark via the Hive JDBC driver, which targets the Spark Thrift Server. The Hive JDBC driver is bundled with Tessallite.

Parameter Value
Host Hostname or IP of the Spark Thrift Server
Port 10000 (default Thrift Server port)
Database Hive/Spark database name
Authentication None or Kerberos

Tested with: Apache Spark 3.x Thrift Server.

Kerberos authentication: Kerberos requires additional configuration (keytab path, principal). Refer to the Tessallite deployment guide for Kerberos setup details.

Databricks SQL Warehouse: Databricks does not use the Hive JDBC driver. For Databricks, download the Databricks JDBC driver and configure it as a custom JDBC source. Connection parameters follow the Databricks JDBC driver documentation.
