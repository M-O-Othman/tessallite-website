Aggregates Not Building

What this covers

Diagnosing situations where aggregates fail to build, are stuck in a building state, or never appear. Aggregates are built by the Tessallite Scheduler service. Most failures trace back to a connectivity, permission, or configuration problem.

Symptom reference

Symptom Likely cause Resolution
Aggregate stuck in "Building" status for more than 30 minutes Scheduler service has crashed or is not running Check the Scheduler service status: docker compose ps scheduler. If it shows Exit or Restarting, read the logs: docker compose logs --tail=50 scheduler. Restart if needed: docker compose restart scheduler.
Aggregate status shows "Error" Build query failed — see error details in Model Builder Open Model Builder → Health tab → find the "Aggregate build failed" entry → expand it for the error message. See the sub-causes below.
"Error: write permission denied on target schema" The database user used by Tessallite cannot write to the aggregate target schema Grant CREATE TABLE, INSERT, and DROP TABLE on the target schema to the database user configured in the data source connection. Verify with your DBA.
"Error: source query timeout" The source query for the aggregate build exceeded the query timeout Either increase the query timeout in Workspace Settings, or reduce the aggregate grain (fewer dimensions = smaller query).
"Error: source connection refused" The source data source is not reachable at build time Verify the source connection parameters in the project data source settings. Confirm the source database is running and accessible from the Tessallite host.
No new aggregates appear after running the AI Optimizer The Optimizer's miss log is empty — no queries have been captured yet Run several queries through a BI tool first. The Optimizer needs query history to identify patterns. Then re-open the Optimizer to see suggestions.
AI Optimizer suggests nothing even with query history Score threshold is too high — no patterns exceed the minimum score Reduce the OPTIMIZER_SCORE_THRESHOLD environment variable and restart the Optimizer service.
An existing aggregate disappears Optimizer retired an unused aggregate If the aggregate is still needed, re-create it manually in Model Builder, or lower the retirement threshold in Optimizer settings.

View Scheduler logs

To view Scheduler logs in a local Docker Compose deployment:

docker compose logs -f scheduler

The -f flag streams new log lines as they appear. Press Ctrl+C to stop. Look for lines containing ERROR or FATAL for the root cause.

For Cloud Run or other managed deployments, access logs through the platform's log viewer and filter by the scheduler service name.

Common permission grant

For PostgreSQL target schemas, run the following as a database superuser to grant the required permissions:

GRANT USAGE ON SCHEMA target_schema TO tessallite_user;
GRANT CREATE ON SCHEMA target_schema TO tessallite_user;
GRANT INSERT, SELECT, DROP ON ALL TABLES IN SCHEMA target_schema TO tessallite_user;

Replace target_schema with the schema configured as the aggregate write target, and tessallite_user with the database user configured in the Tessallite data source connection.
