Common Errors

What this covers

A reference table of error messages returned by Tessallite services and the Gateway, grouped by category. For step-by-step diagnostic guides, see the linked troubleshooting articles.

Connection errors

These errors appear when a BI tool or API client cannot establish a connection to Tessallite.

Error message Likely cause Resolution
FATAL: database "X" does not exist The value in the Database field is not a valid workspace slug Verify the workspace slug with your Tenant Admin. The slug is case-sensitive. Example: acme, not Acme or acme_db.
FATAL: password authentication failed for user "X" Wrong username or password Confirm you are using Tessallite credentials (email and password), not the credentials for the underlying data source. Reset via the Admin panel if needed.
Connection refused (port 5433) Gateway service is not running, or wrong host/port Ask your System Admin to check the Gateway service: docker compose ps gateway. Confirm the hostname and port are correct.
SSL SYSCALL error: EOF detected Server requires SSL but the client is not configured for it Append ?sslmode=require to the JDBC URL, or enable SSL in your client connection settings.

Query errors

These errors appear in BI tool error dialogs or in the API response body when a query fails.

Error message Likely cause Resolution
ERROR: column "X" does not exist A dimension or measure in the model references a column that has been renamed or removed in the source table (schema drift) Open Model Builder → Health tab → look for "Schema drift detected" → update the model to reference the correct column → re-publish and rebuild aggregates.
ERROR: Query timeout exceeded The query ran longer than the workspace query timeout setting Increase the query timeout in Workspace Settings. For recurring slow queries, ask your Modeller to build a matching aggregate so the query is served from a pre-built summary.
ERROR: permission denied for schema "X" The Tessallite database user lacks read permission on the source schema Grant USAGE on the schema and SELECT on the required tables to the database user configured in the data source connection.

Aggregate errors

These errors appear in the Model Builder Health tab or in Scheduler logs when an aggregate build fails.

Error message Likely cause Resolution
Aggregate build failed: write permission denied The Scheduler cannot write pre-aggregated results to the target schema Grant CREATE TABLE, INSERT, and DROP TABLE on the target schema to the Tessallite database user. See Aggregates Not Building for the full grant command.
Aggregate build failed: source timeout The source query for the aggregate build exceeded the query timeout Reduce the aggregate grain (use fewer dimensions) or increase the source query timeout in Workspace Settings.
Aggregate retired The AI Optimizer identified the aggregate as unused and removed it Re-create the aggregate manually in Model Builder if it is still needed. To prevent future retirements, lower the retirement threshold in the Optimizer settings.
