Excel XMLA Connection Guide

What this covers

This is the detailed connection reference for Microsoft Excel connecting to Tessallite via the XMLA endpoint. Tessallite exposes an Analysis Services-compatible XMLA interface on port 8080. Excel's Analysis Services connector uses this interface to query models as if they were OLAP cubes.

For a shorter introduction, see Connect Excel via XMLA.

XMLA endpoint details

Parameter Value Notes
URL http://HOST:8080/xmla Replace HOST with the Tessallite Gateway hostname. Must include the /xmla path.
Authentication HTTP Basic Username and password of a Tessallite user account.
Catalog Workspace slug (e.g., acme) Case-sensitive. Obtain from your Tenant Admin.
Protocol XMLA 1.1 Standard Analysis Services protocol.
Cube / Perspective Model name within the project Shown in the catalog browser once you connect.

Connect Excel to Tessallite

  1. Open Excel.
  2. Go to Data → Get Data → From Other Sources → From Analysis Services.
  3. In the Server name field, enter the full XMLA URL: http://HOST:8080/xmla
  4. Under Log on credentials, select Use the following User Name and Password.
  5. Enter your Tessallite username (email address) and password.
  6. Click Next.
  7. In the Select the database dropdown, choose your workspace slug (the catalog).
  8. In the cube list, select the model name you want to query.
  9. Click Next, then Finish.
  10. In the Import Data dialog, select PivotTable Report and click OK.

A PivotTable is inserted in the worksheet. The field list on the right shows the dimensions and measures from the selected model.

Create a PivotTable

Drag dimensions to the Rows or Columns area and measures to the Values area. Excel sends MDX queries to Tessallite, which routes them to the fastest available source.

Refresh data

To refresh the data in a connected PivotTable, right-click anywhere in the PivotTable and select Refresh. This sends a new query to Tessallite and updates the displayed values.

To configure automatic refresh, go to Data → Queries & Connections, right-click the connection, and select Properties. Under the Usage tab, enable Refresh every N minutes or Refresh data when opening the file.

Manage connection properties

  1. Go to Data → Queries & Connections.
  2. Right-click the Tessallite connection and select Properties.
  3. On the Definition tab you can modify the connection string and command text.
  4. On the Usage tab you can set refresh intervals and control connection behavior when the file opens.

Troubleshooting

Problem Likely cause Fix
Cannot connect / "Unable to connect to the server" Wrong URL format or port blocked by firewall Verify the URL is exactly http://HOST:8080/xmla. Do not use HTTPS unless SSL is configured. Test port access: curl -v http://HOST:8080/xmla
"Catalog not found" or no catalog in dropdown Wrong workspace slug Check the workspace slug with your Tenant Admin. Slug is case-sensitive.
"Authentication failed" / "The user name or password is incorrect" Wrong credentials Reset your Tessallite password via the Admin panel and try again.
"No cubes found" / cube list is empty No published model in the selected project Ask your Modeller to verify the model is saved and published in Model Builder.
Excel cached a bad connection Stale connection cache Go to Data → Queries & Connections, right-click the connection, and select Delete. Reconnect from scratch.

The XMLA URL must include the /xmla path segment. Entering only http://HOST:8080 will not work.
