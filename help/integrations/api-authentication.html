API Authentication

What this covers

The Tessallite REST API uses HTTP Basic authentication on every request. There are no session tokens, API keys, or OAuth flows. This article explains how to authenticate and what happens when authentication fails.

How authentication works

Each HTTP request to the API must include an Authorization header with HTTP Basic credentials. The credentials are a Tessallite username (email address) and password, base64-encoded in the standard Basic auth format.

Base URL: http://HOST:3000/api/v1

Most HTTP clients (curl, Python requests, Postman) handle the encoding automatically when you supply a username and password.

HTTP Basic auth transmits credentials as base64, not encrypted. Use HTTPS in all production environments to prevent credentials from being intercepted in transit.

Permissions

The permissions available to an API caller depend on the role assigned to the Tessallite user account used for authentication:

  • System Admin: full access to all endpoints, including workspace management.
  • Tenant Admin: access to project and workspace-level endpoints within their tenant.
  • Modeller: access to project, model, and aggregate endpoints.
  • Analyst / Viewer: read-only access; cannot call administrative endpoints.

The System Admin credentials configured in the ADMIN_USER and ADMIN_PASS environment variables have full access to all API endpoints, including workspace creation and deletion.

Example: curl

curl -u username:password http://HOST:3000/api/v1/health

The -u flag passes credentials as HTTP Basic auth. Replace username with your Tessallite email address and password with your Tessallite password.

Example: Python requests

import requests

response = requests.get(
    "http://HOST:3000/api/v1/health",
    auth=("user@example.com", "yourpassword")
)
print(response.json())

The auth parameter in the Python requests library accepts a tuple of (username, password) and sends it as HTTP Basic auth automatically.

Common authentication errors

HTTP status Meaning Resolution
401 Unauthorized No credentials supplied, or credentials are wrong Verify your Tessallite email address and password. Confirm the Authorization header is being sent.
403 Forbidden Credentials are correct but the user's role does not permit the requested endpoint Use an account with the required role. For workspace management, a System Admin account is required.

Security recommendations

  • Always use HTTPS in production. Plain HTTP exposes credentials to network observers.
  • Do not store credentials in source code. Use environment variables or a secrets manager.
  • Use a dedicated service account (a Tessallite user created for automation) rather than a personal account for API access. Assign it the minimum required role.
  • Rotate the service account password if it may have been exposed.
