Use the AI Optimiser

What this covers

The AI Optimiser analyzes query miss patterns against your model, scores potential aggregate configurations by estimated return on investment, and surfaces ranked recommendations. This article explains how to open the Optimiser, how to read the recommendation list, and how to act on each recommendation.

How the Optimiser works

Every time a query reaches the Query Router and no suitable aggregate exists to answer it, the miss is recorded in the query miss log. The Optimiser reads this log, groups misses by their grain (the combination of dimensions and measures requested), and scores each group by the estimated query time that could be saved if an aggregate existed for that grain. The result is a ranked list of candidates ordered from highest to lowest ROI.

The Optimiser requires data in the query miss log before it can make recommendations. At least some queries must have run against the model after it was published. If no queries have run yet, the recommendation list will be empty.

Opening the AI Optimiser

  1. Open your project in Model Builder.
  2. In the Toolbelt on the right side of the screen, click Optimiser.
  3. The AI Optimiser panel opens and displays the current recommendation list.

Reading the recommendation list

Column Description
Grain / Dimensions The set of dimension columns that define this aggregate's level of detail. A finer grain produces a larger aggregate covering more query patterns; a coarser grain produces a smaller aggregate with narrower coverage.
Measures The numeric columns the Optimiser recommends pre-aggregating for this grain. These match the measures that appeared in missed queries.
Query hits The number of times this query pattern was missed since the miss log was last reset. Higher values indicate a frequently recurring pattern.
ROI score A dimensionless score combining query frequency and estimated time saved per query. Use this to decide which aggregates to build first.
Action The available actions for this recommendation: Build, Schedule, or Dismiss.

Actions

Build

Queues the aggregate for immediate construction. The aggregate appears in the Canvas with status Building. When the build completes, the status transitions to Ready and the aggregate becomes available for query routing. The recommendation is removed from the list once the build is queued.

Schedule

Adds the aggregate to the regular build cycle without triggering an immediate build. The aggregate is created in the Canvas with the workspace's default refresh schedule applied. You can adjust the schedule in the aggregate's Drawer or in the Scheduler panel.

Dismiss

Removes the recommendation from the list without creating an aggregate. The underlying miss pattern is not deleted — if the same query pattern continues to miss, the recommendation may reappear in a future scoring cycle.

Re-scoring after acting on a recommendation

After you Build, Schedule, or Dismiss a recommendation, the Optimiser re-scores the remaining candidates and updates the list. Aggregates you have already built or scheduled are excluded from future recommendations, ensuring rankings always reflect current aggregate coverage.

Optimiser vs. manual configuration

Use the Optimiser when you are not certain which queries are slow or which aggregates would have the most impact. The miss log tells you exactly what is being asked and what is not being served from cache.

Use manual aggregate configuration when you know the exact grain you need — for example, when a specific reporting query has a fixed structure and you want direct control over the aggregate definition, refresh schedule, and partitioning.

Both approaches can be used together. The Optimiser handles opportunistic coverage; manual configuration handles known, critical queries.
