Project Settings

What this covers

Every project in Tessallite carries its own configuration drawer — a right-hand pane with nine focused tabs covering connections, LLM provider configurations, and the seven groups that drive the conversational agent.

Why the drawer

Connections, LLM configurations, and the agent's brief, safety policy, knowledge map, and judge configuration are all project-scoped — the same connection or LLM bundle is reused across every model in the project, and the agent answers questions across the project's models. Putting these on a single drawer keeps the model toolbox lean.

Opening the drawer

  1. Open Tenant Administration → Projects.
  2. On the row of the project you want to configure, click the gear icon. The drawer opens from the right.
  3. Use the tab strip across the top to switch sections.

The nine tabs

Tab What it owns Typical edits
Connections Source / target connections (PostgreSQL, BigQuery, Hadoop / Spark Hive Thrift). Add, edit, test, retire connections.
LLM Configurations Named LLM provider bundles. Create one bundle per LLM you want available to the agent and judge.
General Agent master switch and global toggles. Turn the agent on or off.
Identity & Tone Display name, project brief, agent role, tone, brand guidelines, disclosure text. Set the brand-facing voice.
Knowledge Cross-model recipes and per-model context. Manage recipes; edit per-model overviews and example questions.
Safety Safety policy and content rules. Add forbidden topics, content directives.
LLM & Judge Allow-list, answer LLM, judge LLM, judge mode, rubric, block visibility, visibility toggles. Pick LLMs, configure the judge, manage the model allow-list.
Webhook Outbound webhook URL for agent events. Wire turn metadata into a downstream system.
Retention How long agent conversations are kept. Tighten to meet compliance windows.

How saves work

Each agent tab carries one Save button at the bottom; it persists the entire ProjectAgentConfig record in one PUT. Switching between agent tabs preserves the in-progress draft until you save or close the drawer.

Project settings apply to every model in the project. The allow-list and per-model context are also managed from this drawer (LLM & Judge tab and Knowledge tab respectively).

Editing a setting

  1. Open the drawer and switch to the relevant tab.
  2. Adjust the fields you need to change.
  3. Click Save. A success toast confirms the write.
  4. To revert a field, change it back and click Save again.

Best practices

  • Define connections and LLM configs once. Reuse a single bundle and pick it twice — once for answer, once for judge.
  • Keep the project brief short. It is read on every agent turn.
  • Test connections before enabling the agent. A failing connection in the allow-list returns errors at chat time.

Common pitfalls

  • Editing a connection while a refresh runs. Pause aggregate refreshes, edit, then resume.
  • Picking an LLM bundle the project does not have. Add it on the LLM Configurations tab first.
  • Treating the seven agent tabs as a wizard. They are not ordered — save in any order.
