Excel Connection Problems

What this covers

Diagnosing and resolving connection failures when connecting Microsoft Excel to Tessallite via the XMLA endpoint. For the connection setup procedure, see Excel XMLA Connection Guide.

Pre-connection checklist

Before troubleshooting, verify:

  • Port 8080 is reachable from this machine. Test with: curl -v http://HOST:8080/xmla
  • You have the workspace slug (obtain from your Tenant Admin). It is case-sensitive.
  • You have valid Tessallite credentials (email address and password).
  • You know the project name and model name you intend to query.

Symptom reference

Symptom Likely cause Resolution
"We couldn't connect to the Analysis Services server" Wrong URL format The URL must be exactly http://HOST:8080/xmla. Do not omit the /xmla path. Do not use HTTPS unless SSL is explicitly configured on the Tessallite server.
"We couldn't connect to the Analysis Services server" (URL appears correct) Port 8080 blocked by firewall or Gateway service not running Test connectivity from the command line: curl -v http://HOST:8080/xmla. If it times out or is refused, escalate to your System Admin to check the Gateway service and firewall rules.
"The catalog name is invalid" or no catalog appears in the dropdown Wrong workspace slug, or slug entered with wrong case Verify the workspace slug with your Tenant Admin. The slug is case-sensitive. Enter it exactly as provided.
"The user name or password is incorrect" Wrong Tessallite credentials Confirm you are using your Tessallite email address and password (not credentials for the underlying database). Reset your password via the Tessallite Admin panel if needed.
"No cubes were found in the database" or the model list is empty The project has no published model Ask your Modeller to open the project in Model Builder, verify the model is complete, and publish it. A model must be saved and published before it is visible through XMLA.
Data looks stale or values do not match expected figures Aggregate not refreshed after source data changed Ask your Modeller to check the aggregate status in Model Builder. If the status is Stale, run a refresh. If no aggregate exists for this query grain, Tessallite is showing raw-source data which may be at a different freshness level.
Excel shows an error after previously working Excel cached a connection with old or invalid settings Clear the cached connection: go to Data → Queries & Connections, right-click the Tessallite connection, and select Delete. Then reconnect from scratch using the steps in the Excel XMLA Connection Guide.

Testing the XMLA endpoint directly

To verify the endpoint is reachable before opening Excel, run this command from a terminal on the same machine as Excel:

curl -v http://HOST:8080/xmla

A response of 200 OK or any HTTP response (even an error from the server) confirms the port is reachable. A timeout or "connection refused" means the port is not reachable and is a network or service issue, not an Excel configuration issue.

If you receive an HTTP response but Excel still cannot connect, the problem is in the URL, credentials, or catalog name entered in the Excel wizard — not in network connectivity.
