Power BI Connection Guide

What this covers

Power BI Desktop can connect to Tessallite using two methods: the PostgreSQL connector (which uses the JDBC-equivalent connection on port 5433) or the Analysis Services XMLA connector (port 8080). Both methods are described here.

DirectQuery mode is recommended for both methods so that Power BI routes queries to Tessallite at report refresh time rather than downloading and caching data locally.

Method 1 — PostgreSQL connector (port 5433)

This method uses the same JDBC endpoint as DBeaver and Tableau. Power BI treats Tessallite as a standard PostgreSQL server.

  1. Open Power BI Desktop.
  2. Click Get Data → Database → PostgreSQL database → Connect.
  3. In the Server field, enter HOST:5433 (include the port).
  4. In the Database field, enter your workspace slug (e.g., acme).
  5. Under Data Connectivity mode, select DirectQuery.
  6. Click OK.
  7. In the credentials dialog, select Database authentication.
  8. Enter your Tessallite username (email address) and password.
  9. Click Connect.
  10. In the Navigator, select the tables or views to include in the report.
  11. Click Load (to add to the model directly) or Transform Data (to open Power Query for additional shaping).

Method 2 — XMLA / Analysis Services connector (port 8080)

This method connects to the XMLA endpoint and surfaces the Tessallite model as a multidimensional cube with dimensions and measures.

  1. Open Power BI Desktop.
  2. Click Get Data → Online Services → Analysis Services → Connect.
  3. In the Server field, enter the full XMLA URL: http://HOST:8080/xmla
  4. Select Connect live (recommended) or Import.
  5. Click OK.
  6. In the credentials dialog, select Basic authentication.
  7. Enter your Tessallite username and password.
  8. Click Connect.
  9. Select the catalog (workspace slug) from the dropdown.
  10. Select the model name.
  11. Click OK.

DirectQuery vs Import mode

Mode How it works Recommendation
DirectQuery Power BI sends SQL or MDX queries to Tessallite at report open and refresh time. Tessallite routes each query to the optimal source. Recommended for large datasets and real-time data. Retains all acceleration benefits.
Import Power BI downloads the full result set and caches it in the .pbix file. Queries run against the local cache, not Tessallite. Suitable only for small, infrequently changing datasets. Loses live routing and acceleration benefits.

DirectQuery mode is required to benefit from Tessallite's aggregate routing. In Import mode, all acceleration happens during the import step only, and the cached data goes stale between refreshes.

Power BI Gateway for scheduled refresh

If the Tessallite Gateway is on a private network (not accessible from the internet), Power BI Service cannot reach it directly for scheduled refresh. In this case, install an on-premises data gateway on a machine that can reach the Tessallite host. Configure the gateway in Power BI Service and associate the dataset with it.

For Gateway setup, see the Microsoft Power BI Gateway documentation.

Troubleshooting

Symptom Likely cause Resolution
Cannot connect (PostgreSQL method) Gateway not running or wrong host/port Verify the Gateway service is running. Confirm SERVER field includes the port: HOST:5433.
Authentication failed Wrong credentials Use Tessallite email and password. Use Database auth type, not Windows.
Cannot connect (XMLA method) Wrong URL or port 8080 not reachable Verify URL is http://HOST:8080/xmla. Test with curl -v http://HOST:8080/xmla.
Scheduled refresh fails in Power BI Service Tessallite host is on a private network Install and configure an on-premises data gateway on a machine with network access to the Tessallite host.
