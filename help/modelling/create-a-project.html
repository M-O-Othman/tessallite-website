Create a Project

What this covers

A project is the top-level container within a workspace. It connects a workspace to exactly one data source and holds exactly one model. This article explains what a project is, who can create one, the seven-step creation flow, and what you can configure after creation.

Before you start

  • You must be signed in as a Tenant Admin or a Modeller with project-creation permission granted by a Tenant Admin.
  • You need the connection details for the data source you want the project to use (host, credentials, or service account). You can enter these during creation or update them later in the project settings.

What a project contains

Within the multi-tenant hierarchy — Platform, Workspace, Project, Model — a project sits below the workspace. Each project:

  • Connects to exactly one data source (PostgreSQL, BigQuery, or Hadoop/Spark).
  • Contains exactly one model, built in the Model Builder.
  • Has its own access control: team members can be granted Modeller or Analyst roles scoped to that project.

Multiple projects can exist within the same workspace, each pointing to a different (or the same) data source.

Creating a project

  1. Sign in at port 3000 as a Tenant Admin or Modeller.
  2. From the workspace dashboard, click New Project.
  3. Enter a Project name. Use a name that identifies the data domain (for example, Sales Analytics or Logistics 2026).
  4. Enter an optional Description. This is visible to all workspace members in the project list.
  5. Select the Data source type: PostgreSQL, BigQuery, or Hadoop/Spark Thrift Server.
  6. Enter the connection parameters for the selected source type. See Add a Data Source for the full parameter reference.
  7. Click Test Connection to verify the parameters. A success message confirms the gateway can reach the source. If the test fails, check the error message and correct the parameters before continuing.
  8. Click Create Project. The project is created and appears in the workspace project list. The Model Builder opens automatically with an empty model ready to populate.

The Test Connection step does not save the project. If you close the browser before clicking Create Project, no project is created and you must start again.

After creation

Once a project exists, the Model Builder is available immediately. The model is empty — no tables, joins, dimensions, or measures have been added yet. Start by adding tables; see Add Tables to a Model.

Project settings after creation

To access project settings, open the project and click the Settings tab. The following fields are editable after creation:

Setting What it controls Who can change it
Name The display name shown in the workspace project list and in BI tool schema listings. Tenant Admin, Modeller
Description Free-text description shown in the project list. Tenant Admin, Modeller
Data source The connection parameters and source type for the project. Changing the source type after tables have been added will invalidate the model; all tables must be re-added from the new source. Tenant Admin
