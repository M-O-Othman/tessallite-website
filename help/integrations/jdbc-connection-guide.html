JDBC Connection Guide

What this covers

Tessallite exposes a PostgreSQL wire-protocol endpoint on port 5433. Any JDBC client that supports the standard PostgreSQL driver can connect to it — including DBeaver, Tableau, and any tool using the psycopg2 or JDBC PostgreSQL driver. No Tessallite-specific driver is required.

This article is a detailed connection reference. If you want a shorter walkthrough, see Connect a BI Tool via JDBC.

Connection parameters

Parameter Value Notes
Host Hostname or IP of the Tessallite Gateway Use localhost for local installs. Obtain from your System Admin for cloud deployments.
Port 5433 Fixed. Not the standard PostgreSQL port (5432).
Database Workspace slug (e.g., acme) Case-sensitive. This is not a real database name — it routes to the correct workspace. Obtain the slug from your Tenant Admin.
Username Your Tessallite username (email address)
Password Your Tessallite password
SSL Optional Add ?sslmode=require to the JDBC URL if the server requires SSL. Not required for local installs.
Driver class org.postgresql.Driver Standard PostgreSQL JDBC driver. No Tessallite-specific driver.

JDBC URL format

jdbc:postgresql://HOST:5433/WORKSPACE_SLUG

Example with a real hostname and workspace slug:

jdbc:postgresql://analytics.example.com:5433/acme

With SSL required:

jdbc:postgresql://analytics.example.com:5433/acme?sslmode=require

The "database" field in your client must contain the workspace slug, not a real database name. If you enter anything else, the Gateway returns FATAL: database "X" does not exist.

Connect with DBeaver

  1. Open DBeaver.
  2. Click New Database Connection (the plug icon in the toolbar, or Database menu).
  3. Select PostgreSQL from the connection type list and click Next.
  4. Fill in the fields:
    • Host: your Gateway hostname
    • Port: 5433
    • Database: your workspace slug
    • Username: your email address
    • Password: your Tessallite password
  5. Click Test Connection. A "Connected" dialog confirms success.
  6. Click Finish.
  7. Expand the connection in the left panel. Each published model in the workspace appears as a schema, with its dimensions and measures listed as columns.

Connect with Tableau

  1. Open Tableau Desktop.
  2. In the Connect pane, under To a Server, select PostgreSQL.
  3. Enter the Gateway hostname in the Server field.
  4. Enter 5433 in the Port field.
  5. Enter the workspace slug in the Database field.
  6. Enter your credentials in Username and Password.
  7. Click Sign In.
  8. The data source pane populates with the available tables from the workspace model.

Driver note

Tessallite uses the PostgreSQL wire protocol. You do not need a special driver. Use the standard org.postgresql.Driver (JDBC) or the psycopg2 library (Python). Any tool that can connect to a standard PostgreSQL server on a custom port can connect to Tessallite without modification.

Troubleshooting

Symptom Likely cause Resolution
Connection refused on port 5433 Gateway not running or wrong host Ask your System Admin to verify the Gateway service is up. Confirm the hostname is correct.
FATAL: database "X" does not exist Wrong workspace slug in the Database field Verify the slug with your Tenant Admin. Slug is case-sensitive.
Authentication failed Wrong username or password Confirm you are using your Tessallite email and password, not credentials for the underlying data source.
SSL error / SSL required Server requires SSL, client not configured for it Append ?sslmode=require to the JDBC URL.
No tables visible after connecting No published models in the workspace A Modeller must create and publish at least one model before tables appear.
