Screenshot of the Model Builder Pocket Drawer in create mode, opened from the Pocket Tables panel's New Pocket button. Right-anchored 720 px drawer with three tabs in the header — Query (active, underlined), Schedule, Advanced (Advanced disabled until the pocket is saved). The Query tab body shows: a Target dropdown labelled "Target (where the pocket is stored)" set to "public"; an info hint card reading "A pocket is a materialised slice of its model. Reference the model by slug — e.g. SELECT * FROM <model_slug> WHERE <dimension> = '...'. Joins, GROUP BY, aggregates and projections other than * are not allowed at definition time."; a multi-line SqlQueryEditor with placeholder "SELECT * FROM <model_slug> WHERE ..."; below the editor, Validate and Dry Run buttons (left). Footer carries Cancel (left) and Save (right, disabled until the SQL is provided). Behind the drawer the Canvas shows the model's dimension tables. Toolbelt visible on the left with the Pocket Tables icon highlighted. Dimensions: 1280x800px.
